## ?? Skills System (IA + Automatizaciуn)

El repositorio incluye un sistema de **Skills de IA** para trabajar por capas: auditorнa, sincronizaciуn documental, validaciуn de release, mantenimiento SEO/LLM, chequeo mobile y actualizaciуn integral del portafolio.

| Skill | Cuбndo usarlo |
|---|---|
| [`portfolio-full-update`](.agents/skills/portfolio-full-update/SKILL.md) | Cuando cambian CV, experiencia, proyectos, PDFs o quieres refrescar el portafolio completo de punta a punta |
| [`portfolio-consistency-audit`](.agents/skills/portfolio-consistency-audit/SKILL.md) | Cuando quieres detectar contradicciones entre `index.html`, `api/v1/`, `README`, docs, wiki, SEO y artefactos pъblicos |
| [`portfolio-release-guard`](.agents/skills/portfolio-release-guard/SKILL.md) | Antes de commit/push o release para validar lint, build, integridad de `dist/` y rutas pъblicas crнticas |
| [`portfolio-doc-sync`](.agents/skills/portfolio-doc-sync/SKILL.md) | Cuando cambiу cуdigo, build, API o arquitectura y hace falta alinear README, manuales y wiki sin rehacer todo |
| [`portfolio-seo-llm-maintainer`](.agents/skills/portfolio-seo-llm-maintainer/SKILL.md) | Cuando cambian rutas, metadatos, discoverability, `llm.txt`, `robots.txt`, `sitemap.xml` o claims SEO/IA |
| [`portfolio-mobile-wrapper-check`](.agents/skills/portfolio-mobile-wrapper-check/SKILL.md) | Cuando cambian web/PWA/assets y quieres confirmar que Android/iOS siguen alineados con Capacitor |

### Uso recomendado

* Di *"actualicй mi CV"* o *"quiero refrescar el portafolio completo"* para disparar `portfolio-full-update`.
* Di *"audita consistencia del portafolio"* para usar `portfolio-consistency-audit`.
* Di *"verifica que puedo publicar sin romper nada"* para usar `portfolio-release-guard`.
* Di *"sincroniza la documentaciуn con los cambios"* para usar `portfolio-doc-sync`.
* Di *"actualiza la capa SEO y llm.txt"* para usar `portfolio-seo-llm-maintainer`.
* Di *"revisa si el wrapper mobile sigue sano"* para usar `portfolio-mobile-wrapper-check`.

### Orden sugerido

1. `portfolio-consistency-audit` para detectar deriva.
2. `portfolio-doc-sync` o `portfolio-seo-llm-maintainer` segъn el tipo de cambio.
3. `portfolio-mobile-wrapper-check` si tocaste PWA, manifest o assets.
4. `portfolio-release-guard` antes de subir.
5. `portfolio-full-update` cuando el cambio sea transversal y afecte varias capas.
